# DHU Guidance 管理ポータル 運用マニュアル

## 1. はじめに

管理ポータルは、DHU大学院 履修ガイダンスポータルのコンテンツ（FAQ・キャリアマップ・設定）をブラウザ上で編集するためのツールです。

**開発サーバーでのみ動作**します（本番ビルドには含まれません）。

---

## 2. 起動方法

### 前提条件

- Node.js がインストールされていること
- プロジェクトの依存関係がインストール済み（`npm install`）

### 起動コマンド

```bash
npm run admin
```

ブラウザが自動で `http://localhost:5173/admin.html` を開きます。
（ポートが使用中の場合は 5174、5175... に自動変更されます）

> **補足**: `npm run dev` で起動した場合も `/admin.html` にアクセスすれば管理ポータルを利用できます。

---

## 3. 管理画面の構成

管理ポータルは以下の **4つのタブ** で構成されています。

| タブ | 内容 |
|------|------|
| FAQ編集 | よくある質問の追加・編集・削除・並べ替え |
| キャリアマップ | キャリアパスの追加・編集・削除、推奨科目の管理 |
| 設定・年度 | 年度表示やスライドURL等の設定、シラバス同期 |
| データ検証 | 全データの整合性チェック |

---

## 4. FAQ編集

### 4.1 一覧画面

- 画面上部の **検索ボックス** でキーワード検索ができます
- **カテゴリ** ドロップダウンで表示を絞り込めます
- 各FAQカードには ID、カテゴリ、質問文、回答（抜粋）が表示されます

### 4.2 新規追加

1. 右上の **「+ 新規追加」** ボタンをクリック
2. 編集モーダルが開きます
3. 以下を入力：
   - **カテゴリ**: ドロップダウンから選択
   - **質問**: 質問文を入力
   - **回答**: 回答文を入力
4. **「保存する」** をクリック

### 4.3 編集

1. 対象FAQの **「編集」** ボタンをクリック
2. 編集モーダルで内容を修正
3. **「保存する」** をクリック

### 4.4 並べ替え

- 各FAQカードの **▲▼ボタン** で表示順を変更できます
- 順番を変えるとすぐに保存されます

### 4.5 削除

1. 対象FAQの **「削除」** ボタンをクリック
2. 確認ダイアログで **OK** を選択

### 4.6 リンクの書き方

回答文中にリンクを埋め込む場合は、以下の書式を使ってください：

```
[表示テキスト](URL)
```

例：
```
詳細は[事務局ウェブサイト](https://www.dhw.ac.jp)をご確認ください。
```

## 5. キャリアマップ編集

### 5.1 一覧画面

- 画面上部の **カテゴリボタン** で表示を絞り込めます（クリエイティブ、テクノロジー等）
- 各カードにはアイコン、カテゴリ、タイトル、説明、推奨科目数が表示されます

### 5.2 新規追加

1. 右上の **「+ 新規追加」** ボタンをクリック
2. 以下を入力：
   - **タイトル**: キャリアパス名（例：「Webデザイナー」）
   - **アイコン**: 絵文字1文字（例：🎨）
   - **カテゴリ**: ドロップダウンから選択
   - **説明**: キャリアパスの説明文
3. 推奨科目を追加（後述）
4. **「保存する」** をクリック

### 5.3 推奨科目の追加

キャリアパス編集画面の **推奨科目** セクションで：

1. **「+ 科目を追加」** をクリック
2. **科目選択モーダル** が開きます
3. 検索ボックスで科目名・教員名・キーワードを検索
4. 対象科目をクリックして選択
5. **推奨理由** を入力（例：「Web制作の基礎スキル習得に」）
6. **「追加」** をクリック（またはEnterキー）

> **注意**: 必修科目は選択肢に表示されません（全員が履修するため）。
> 既に追加済みの科目は「追加済み」として表示されます。

### 5.4 推奨科目の編集・削除

- 各推奨科目の **推奨理由** は直接テキストボックスで編集できます
- **✕ボタン** で推奨科目を削除できます
- 変更後は **「保存する」** を忘れずにクリックしてください

### 5.5 削除

1. 対象キャリアパスの **「削除」** ボタンをクリック
2. 確認ダイアログで **OK** を選択

## 6. 設定・年度

### 6.1 基本設定

| 項目 | 説明 | 例 |
|------|------|-----|
| 年度表示 | ヘッダー右上に表示されるバッジ | `2026年度` |
| ガイダンスタイトル | ガイダンスタブに表示されるタイトル | `2026年度前期 履修ガイダンス` |

### 6.2 Google連携

| 項目 | 説明 |
|------|------|
| スライドEmbed URL | ガイダンスタブに埋め込むGoogle SlidesのURL |
| シラバスSpreadsheet ID | シラバス同期に使用するSpreadsheetのID |

> **スライドEmbed URLの取得方法**:
> 1. Google Slidesでプレゼンテーションを開く
> 2. メニュー → ファイル → ウェブに公開
> 3. 「埋め込み」タブを選択
> 4. 表示されるiframeタグの `src="..."` の中身をコピー

設定変更後は **「設定を保存する」** ボタンをクリックしてください。

### 6.3 シラバス同期

Google Spreadsheetからシラバスデータを最新の状態に更新します。

1. **「シラバスを同期する」** ボタンをクリック
2. 同期が完了すると **差分レポート** が表示されます：
   - **新規追加**: 新しく追加された科目
   - **削除**: 削除された科目
   - **変更あり**: 内容が変更された科目とその変更フィールド
   - **変更なし**: 前回と差がない場合

> **注意**: シラバス同期にはGoogle APIの認証情報（サービスアカウント）が必要です。
> `credentials.json` がプロジェクトルートに存在することを確認してください。

## 7. データ検証

全データの整合性を自動チェックします。

### 7.1 実行方法

1. **「検証を実行」** ボタンをクリック
2. 結果がサマリー（合格・警告・エラーの件数）と詳細リストで表示されます

### 7.2 チェック項目

| カテゴリ | チェック内容 |
|----------|-------------|
| FAQ | IDの重複がないか |
| FAQ | 全エントリに必須フィールド（id, cat, q, a）があるか |
| FAQ | 使用カテゴリがFAQ_CATSに登録されているか |
| キャリア | 全パスに必須フィールド（title, icon, category, desc）があるか |
| キャリア | カテゴリがCARRER_CATEGORIESに登録されているか |
| キャリア | 推奨科目IDがシラバスに存在するか |
| キャリア | 推奨科目が0件のパスがないか（警告） |
| 設定 | academicYearが設定されているか |
| 設定 | slidesEmbedUrlが設定されているか（警告） |
| シラバス | IDの重複がないか |

### 7.3 結果の見方

- **合格**（緑）: 問題なし
- **警告**（黄）: 動作に支障はないが確認推奨
- **エラー**（赤）: 修正が必要

> **推奨**: データ編集後は必ず検証を実行し、エラーがないことを確認してください。

## 8. 自動バックアップ

データを保存するたびに、変更前のファイルが `.backup/` ディレクトリに自動保存されます。

```
.backup/
  faqs.js.2026-03-07T12-30-45-123Z.bak
  careers.js.2026-03-07T12-30-50-456Z.bak
  config.json.2026-03-07T13-00-00-000Z.bak
```

万一データを壊してしまった場合、バックアップファイルから復元できます：

```bash
# 例: faqs.js を復元する場合
cp .backup/faqs.js.2026-03-07T12-30-45-123Z.bak src/data/faqs.js
```

## 9. 学生ポータルへの即時反映

管理ポータルでデータを保存すると、ViteのHMR（Hot Module Replacement）により **学生ポータル側もリアルタイムで更新** されます。

確認手順：
1. ブラウザの別タブで `http://localhost:5173/` を開いておく
2. 管理ポータルでFAQを編集・保存する
3. 学生ポータル側が自動的にリロードされ、変更が反映される

## 10. 本番デプロイ

### ビルド

```bash
npm run build
```

`dist/` ディレクトリにビルド結果が出力されます。
管理ポータル（admin.html）は本番ビルドに**含まれません**。

```bash
git add -A && git commit -m "update content"
git push
```

GitHub Actionsが自動でビルド・デプロイを実行します（設定済みの場合）。

## 12. トラブルシューティング

### 管理ポータルにアクセスできない

- `npm run admin` または `npm run dev` でサーバーが起動しているか確認
- URL末尾が `/admin.html` になっているか確認

### 保存に失敗する

- ブラウザのコンソール（F12 → Console）でエラーメッセージを確認
- ファイルの書き込み権限があるか確認
- 入力内容に不正な文字が含まれていないか確認

### シラバス同期に失敗する

- `credentials.json`（Google APIサービスアカウント）が存在するか確認
- Spreadsheet IDが正しいか確認
- Google APIの権限設定を確認（スプレッドシートがサービスアカウントに共有されているか）

### データ検証でエラーが出る

- エラーメッセージに従って該当データを修正
- 例：「推奨科目IDがシラバスに存在しません」→ シラバス同期後に科目IDが変わった可能性。キャリアマップ編集で推奨科目を再設定

### 学生ポータルに反映されない

- 開発サーバーが起動しているか確認
- ブラウザのキャッシュをクリア（Ctrl+Shift+R）
